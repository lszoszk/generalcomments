## P1 · Real bugs to fix

### B1. Tab `<title>` doesn't follow the open document in the reader
File: `tests/docs-reader.spec.ts` · `R8. titleSyncReader`

When you open `#documents/<doc-id>`, the browser tab title still shows
whatever paragraph was last activated in the search dossier. First
flagged in `USER_TEST_PLAN_DRAFT.md` and the v17 synthetic round
report (Issue #2) but never fixed. Fix: have `paintDocReaderBody()`
call `updateDocumentTitle()` (or set `document.title` directly).

### B2. Dossier-toolbar bookmark button doesn't visually flip "on"
File: `tests/dossier-toolbar.spec.ts` · `D1. saveToggles`

`#ws-bookmark` keeps `class="dossier-tool "` after click — the `.on`
class never lands, even though the underlying `bmToggle()` does write
to localStorage and the badge counter does update. The result-row
☆ marks and docs-reader per-paragraph ☆ buttons DO toggle correctly,
so the bug is specific to the dossier-toolbar #ws-bookmark element.
Likely cause: `paintDossier()` paint after `bmToggle` runs but the
`bmHas(para.id)` lookup doesn't see the new state in time, OR the
paint result string is stale.

## P2 · Accessibility (4 views, all skipped)

File: `tests/a11y.spec.ts` — known violations on every top-level view:
- **color-contrast**: `.dim` text and `.badge-preview/jur` over
  light paper-tone backgrounds drop below WCAG AA 4.5:1. Affects
  search/about/workspace.
- **aria-required-attr**: year-range sliders (`#year-min` / `#year-max`)
  lack required ARIA attributes for their `role="slider"` semantics.
- **target-size**: result-row workspace marks (☆/📌/”/📝) are 24×24 px
  or smaller — WCAG 2.2 wants 24×24 minimum (best practice 44×44).
- **a11y · documents**: timeout — axe-core takes >30s on the 614-row
  rail; tag the rail with `role="navigation"` and the audit can
  short-circuit.

Fix order suggested:
1. Bump target sizes — add `min-width / min-height: 24px` to `.ws-mark*`.
2. Add aria-label / role attrs to the year-range sliders (`role="slider"
   aria-valuemin aria-valuemax aria-valuenow`).
3. Audit `--ink-3 / --ink-4` against `--paper / --paper-2` and bump.
4. Wrap `.docs-rail-list` in `role="navigation" aria-label="Documents"`
   so axe doesn't traverse every row.

## P3 · Two flaky tests (real autosave behaviour, brittle harness)

`D4. noteEditor` + `W4. inlineNote` — autosave-on-blur tests.

Manual testing confirms autosave works (notes survive page reloads
in normal use). The Playwright harness's "click another element to
blur" approach occasionally doesn't trigger the underlying `blur`
event listener in headless mode. Fix: replace `await page.locator(X).click()`
with `await ta.blur()` and add a 1.0s post-save wait. Drop the
`.fixme` once stable.
